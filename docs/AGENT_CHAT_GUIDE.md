# AI Agent Chat 가이드

## 📋 개요

AgenticCP의 AI Agent Chat은 자연어로 멀티클라우드 플랫폼을 제어할 수 있는 혁신적인 기능입니다.

## 🤖 주요 기능

### 1. 자연어 명령 처리

복잡한 UI 조작 없이 자연어로 작업을 수행할 수 있습니다.

**예시:**
- "AWS EC2 인스턴스 목록을 보여줘"
- "이번 달 비용이 가장 많이 나온 리소스는?"
- "us-east-1 리전에 t3.micro 인스턴스 배포해줘"
- "최근 1시간 동안의 알림 보여줘"

### 2. 실시간 대화

Agent와 실시간으로 대화하며 작업을 진행할 수 있습니다.

### 3. 명령 실행 결과

실행된 명령과 결과를 시각적으로 확인할 수 있습니다.

### 4. 대화 히스토리

이전 대화 내용을 저장하고 참조할 수 있습니다.

## 🚀 사용 방법

### 1. Agent Chat 열기

헤더의 "AI Agent" 버튼을 클릭하여 채팅 패널을 엽니다.

```
상단 헤더 > AI Agent 버튼 클릭
```

### 2. 메시지 전송

- 텍스트 입력창에 명령을 입력합니다
- **Enter**: 메시지 전송
- **Shift + Enter**: 줄바꿈

### 3. 명령 팔레트 사용

자주 사용하는 명령을 빠르게 실행할 수 있습니다.

## 💡 명령어 예시

### Cloud Management

```
# 리소스 조회
"AWS EC2 인스턴스 목록 보여줘"
"GCP의 활성 VM 개수 알려줘"
"Azure 리소스 그룹 목록"

# 리소스 상태 확인
"stopped 상태의 인스턴스 찾아줘"
"CPU 사용률이 높은 인스턴스는?"
```

### Cost Management

```
# 비용 조회
"이번 달 AWS 비용 보여줘"
"지난 달 대비 비용 증가율은?"
"비용이 가장 많이 나오는 서비스는?"

# 비용 분석
"AWS 비용을 리전별로 분석해줘"
"비용 절감 방안 제안해줘"
```

### Monitoring

```
# 알림 조회
"최근 1시간 동안의 알림 보여줘"
"심각한 알림만 필터링해서 보여줘"

# 메트릭 조회
"지난 24시간 동안의 CPU 사용률 추이"
"메모리 사용량이 80% 이상인 인스턴스는?"
```

### Resource Orchestration

```
# 리소스 배포
"us-east-1에 t3.micro EC2 인스턴스 배포해줘"
"새로운 로드밸런서 생성해줘"

# 스케일링
"인스턴스 2개 추가해줘"
"Auto Scaling 그룹 크기 조정"
```

### Security

```
# 사용자 관리
"활성 사용자 목록 보여줘"
"관리자 권한을 가진 사용자는?"

# 권한 확인
"내 권한 확인해줘"
"특정 사용자의 역할 보여줘"
```

## 🎯 고급 기능

### 1. 컨텍스트 유지

대화 중 이전 대화를 참조하여 연속적인 작업을 수행할 수 있습니다.

```
사용자: "AWS EC2 인스턴스 목록 보여줘"
Agent: [목록 표시]

사용자: "이 중에서 CPU 사용률이 가장 높은 것은?"
Agent: "i-1234567890이 CPU 85%로 가장 높습니다"

사용자: "이 인스턴스 상세 정보 보여줘"
Agent: [상세 정보 표시]
```

### 2. 복합 명령

여러 작업을 한 번에 요청할 수 있습니다.

```
"AWS와 GCP의 실행 중인 인스턴스 개수를 비교해서 보여줘"
"이번 달 비용을 분석하고 최적화 방안을 제안해줘"
```

### 3. 조건부 작업

조건을 지정하여 작업을 수행할 수 있습니다.

```
"CPU 사용률이 90% 이상인 인스턴스가 있으면 알려줘"
"비용이 $1000을 초과하면 경고해줘"
```

## 🔐 보안 및 권한

### 권한 확인

Agent는 사용자의 권한을 자동으로 확인하여 허용된 작업만 수행합니다.

```
사용자 (Viewer 권한): "인스턴스 삭제해줘"
Agent: "죄송합니다. 리소스 삭제 권한이 없습니다."
```

### 민감한 작업

중요한 작업은 추가 확인을 요청합니다.

```
사용자: "프로덕션 데이터베이스 삭제해줘"
Agent: "중요한 작업입니다. 정말 삭제하시겠습니까? (예/아니오)"
```

## 📊 명령 실행 결과

### 성공

```
✅ 명령이 성공적으로 실행되었습니다
📊 결과: [데이터 표시]
```

### 실패

```
❌ 명령 실행에 실패했습니다
🔍 원인: [오류 메시지]
💡 제안: [해결 방법]
```

## 🎨 UI 구성

### 채팅 패널

```
┌─────────────────────────────┐
│  🤖 AI Agent         [ X ]  │  ← 헤더
├─────────────────────────────┤
│                             │
│  [대화 내용]                │  ← 메시지 영역
│                             │
│                             │
├─────────────────────────────┤
│  메시지 입력...             │  ← 입력 영역
│  [전송 버튼]                │
└─────────────────────────────┘
```

### 메시지 유형

1. **사용자 메시지**: 오른쪽 정렬, 파란색
2. **Agent 메시지**: 왼쪽 정렬, 흰색
3. **시스템 메시지**: 중앙 정렬, 주황색

## 💡 팁과 트릭

### 1. 구체적으로 질문하기

```
❌ "리소스 보여줘"
✅ "AWS의 실행 중인 EC2 인스턴스 목록을 보여줘"
```

### 2. 시간 범위 지정

```
"지난 7일간의 비용 추이 보여줘"
"오늘 발생한 알림 목록"
```

### 3. 필터링 조건 사용

```
"CPU 사용률이 80% 이상인 인스턴스만"
"tag:environment=production인 리소스만"
```

### 4. 정렬 요청

```
"비용이 높은 순서대로 보여줘"
"최근 생성된 순으로 정렬해줘"
```

## 🔧 설정

### 대화 내역 관리

- **삭제**: 상단의 휴지통 아이콘 클릭
- **새 대화**: 대화 삭제 후 자동으로 새 세션 시작

### 알림 설정

Agent의 응답 알림을 설정할 수 있습니다.

```
설정 > Agent Chat > 알림 활성화
```

## 📱 단축키

| 키 | 동작 |
|----|------|
| `Cmd/Ctrl + K` | Agent Chat 열기/닫기 |
| `Enter` | 메시지 전송 |
| `Shift + Enter` | 줄바꿈 |
| `Esc` | Agent Chat 닫기 |
| `Cmd/Ctrl + L` | 대화 내역 삭제 |

## 🐛 문제 해결

### Agent가 응답하지 않을 때

1. 네트워크 연결 확인
2. 백엔드 서버 상태 확인
3. 브라우저 콘솔에서 오류 확인

### 명령이 이해되지 않을 때

더 구체적으로 질문하거나 명령 팔레트의 예시를 참고하세요.

### 권한 오류

관리자에게 필요한 권한을 요청하세요.
